## 1. 产品定位

### 1.1 一句话

**Rentimate** 帮房东在一个 App 内完成收支确认、租客管理、交接留痕与财务统计，减少 Excel 和聊天记录分散导致的漏收、错记、扯皮风险。

### 1.2 要解决的问题

| 痛点 | MVP 解法 |
|------|----------|
| 什么时候该收钱、收多少、是否已收不清楚 | 收支模块自动生成应收 + 到期提醒 + 一键确认 |
| 押金收退和额外费用容易漏记 | 押金收取/退还、附加收费、赔偿统一进收支流水 |
| 租客沟通无统一记录 | 租客档案 + 消息触达记录（提醒/收款确认） |
| 交房收房证据分散 | 交接模块集中存合同、清单、照片、文本备注 |
| 月底不清楚赚没赚钱 | 财务模块按时间和房源统计收入/支出/净额 |

### 1.3 MVP 非目标

- 在线支付代扣
- 租客端 App / 小程序
- 电子签章与实名认证
- 多房东协作与子账号权限
- 复杂税务申报自动化

## 2. MVP 核心范围

### 2.1 必须有（P0）

- 账号注册 / 登录（当前仅支持邮箱）
- 房源、租客、租约基础管理
- **收支主流程**
  - 租金应收自动生成（按租约周期）
  - 收租前自动提醒（本地通知/推送）
  - 到账后一键确认收款
  - 确认后自动发租客收款确认消息（MVP 默认 Email）
  - 押金收取确认、押金退还确认
  - 附加收费（如空调费）与赔偿收入确认
  - 支出登记（维修/物业/杂费）
- **租客主流程**
  - 租客档案管理
  - 到期催缴提醒
  - 收款确认通知留痕
  - 押金状态查看
- **交接主流程**
  - 合同文本/附件归档
  - 交房情况说明
  - 收房验房清单
  - 交接照片证据与备注
- **财务主流程**
  - 月度/年度收支统计
  - 房源维度收入/支出/净额统计
  - 核心汇总卡片与趋势图（MVP 可先简版）

### 2.2 应该有（P1）

- 报表导出 CSV
- 租客消息渠道扩展（短信/站内消息）
- 支出分类自定义
- 维修记录模板化
- 全局搜索增强

### 2.3 明确不做（v1.0 之后）

- Android 专属适配
- iPad 专属布局
- 多语言（MVP 仅简体中文）

## 3. 信息架构（4 个核心 Tab）

### 3.1 底部导航

```
┌─────────┬─────────┬─────────┬─────────┐
│  收支   │  租客   │  交接   │  财务   │
│  💸     │  👥     │  📝     │  📊     │
└─────────┴─────────┴─────────┴─────────┘
```

| Tab | 核心职责 | 高频动作 |
|-----|----------|----------|
| 收支 | 作为默认首页，记录确认收支及全局待办提醒 | 确认收款、确认押金收退、登记支出、处理待办 |
| 租客 | 管理租客关系与履约状态 | 查看档案、发送提醒、查看押金状态 |
| 交接 | 存证据与交付记录 | 上传合同、填交接清单、补照片备注 |
| 财务 | 看经营结果与趋势 | 看月报、看房源利润、对比收支 |

### 3.2 收支与财务边界（避免重叠）

- **收支**：负责“记动作”  
  新增应收/应付、到账确认、出账确认、状态变化。
- **财务**：负责“看结果”  
  汇总统计、趋势分析、房源对比、经营概览。

## 4. 核心业务规则

### 4.1 自动租金应收规则

- 创建租约并启用“自动生成应收”后，系统按租期和付租周期生成租金应收单。
- 到达提醒时间前，系统自动提醒房东“即将应收”。
- 到账后房东只需点一次“确认收款”。
- 资金确认后系统更新状态并触发租客确认消息（Email）。

### 4.2 押金规则

- 收到押金时：在**收支**中确认一笔“押金收取”。
- 退还押金时：在**收支**中确认一笔“押金退还”。
- 押金收退都必须形成流水记录，用于后续财务核算和纠纷追溯。

### 4.3 附加收费与赔偿规则

- 空调费、物业代收、损坏赔偿等都作为“其他应收”进入收支。
- 可关联租客、房源与备注，确保账务来源可追溯。

### 4.4 支出规则

- 维修、清洁、物业、税费等作为“应付/支出”记录在收支。
- 财务模块自动汇总进入统计口径。

### 4.5 状态规则（收支单据）

| 状态 | 含义 |
|------|------|
| `pending` | 待确认（未收/未付） |
| `partial` | 部分完成 |
| `completed` | 已确认完成 |
| `overdue` | 逾期未完成 |
| `cancelled` | 已作废 |

## 6. 核心用户流程

### 6.1 登录与全局待办处理（默认落地流程）

1. 用户打开 App 或登录后，直接进入默认落地页「收支」Tab（C-01）。
2. 若有紧急的跨模块待办（如：明天交房任务、即将到期租约），在顶部通过横幅强提醒，点击跳转对应模块。
3. 若有不紧急的常规消息（如：租客已读回执、系统通知），收拢在右上角铃铛。
4. 处理完全局提醒后，页面主体聚焦于财务相关的核心动作。

### 6.2 每月高频流程（收支闭环）

1. 系统自动生成本期租金应收。
2. 到期前自动提醒房东应收金额与应收对象。
3. 房东收到款后在收支中点击“确认收款”。
4. 系统更新状态并自动发送租客确认邮件。
5. 财务模块自动更新收入与净额统计。

### 6.3 支出闭环流程（维修/物业/杂费）

1. 房东在「收支」中通过快捷入口或 C-07 新建支出（选择分类、金额、房源、备注）。
2. 支出进入 C-03 应付列表，状态为 `pending`，可按到期日提醒。
3. 实际付款后在 C-06 点击“确认付款”，支持部分支付与补充凭证。
4. 系统更新支出状态并回写财务统计（支出、净额、分类占比）。

### 6.4 押金闭环流程

1. 签约时确认“押金收取”（收支入账）。
2. 退租结算时确认“押金退还”（收支出账）。
3. 若有扣款，记录扣款原因并关联验房结果。
4. 财务自动反映押金变动，不混淆租金收入。

### 6.5 交接流程

1. 交接创建任务并上传合同。
2. 交房时记录清单与现场图片。
3. 收房时按验房清单逐项确认。
4. 产出结算依据（赔偿/维修/押金退还）。
5. 回写收支与财务统计。

  title, content?, fileUrl?, createdAt

## 8. 通知与消息策略

### 8.1 房东提醒

- 租金到期提醒：当天 / 提前 1 天 / 提前 3 天
- 逾期提醒：每日一次或仅首次
- 交接提醒：交房前、收房前

### 8.2 租客自动消息（MVP）

- 触发时机：房东确认收款后
- 渠道：Email（MVP 默认）
- 内容：收款金额、对应期间、确认时间、备注
- 记录：保存发送结果到消息日志
